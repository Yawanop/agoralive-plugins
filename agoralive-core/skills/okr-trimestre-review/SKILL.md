# okr-trimestre-review — Revue OKR trimestrielle

## Mission

À chaque fin de trimestre, faire un point honnête sur les OKR : qu'est-ce qui a été tenu, qu'est-ce qui a glissé, pourquoi, et qu'est-ce qu'on se fixe pour le suivant.

C'est un rituel critique du pilotage Paul + Julien.

---

## Procédure

### Étape 1 — Récupère les OKR + KR du trimestre écoulé

Depuis les bases Notion :
- 🎯 [Base Objectifs (OKR)](https://www.notion.so/88627a768f894c07a2a7ee5a7044c1cd)
- 🔑 [Base Key Results](https://www.notion.so/4bbcafe6f62147d9a705b27f0ddcd130)

Filtre par trimestre courant (par ex. Q2 2026 = avril-juin).

### Étape 2 — Évalue chaque KR

Pour chaque Key Result :
- Cible visée
- Valeur réelle atteinte
- Score : valeur / cible (entre 0 et 1)

Lecture standard OKR :
- **0.7 à 1.0** : objectif réussi (la barre est volontairement haute)
- **0.4 à 0.7** : objectif partiel — comprendre pourquoi
- **< 0.4** : objectif raté — analyse approfondie

### Étape 3 — Identifie les patterns

- Quels Objectifs ont été surperformés ? (a-t-on visé trop bas ?)
- Quels Objectifs ont été ratés ? (cible irréaliste ? exécution défaillante ? imprévu ?)
- Quels KR étaient mal formulés (pas mesurables) ?

### Étape 4 — Drafte les OKR du trimestre suivant

Méthode :
- 3-5 Objectifs par équipe maximum
- Chaque Objectif a 2-4 KR
- Cibles ambitieuses (atteindre 70% = bon score)
- KR explicitement mesurables (chiffre + date)

Format suggéré :

```
Objectif 1 : <Énoncé qualitatif, inspirant>
• KR 1.1 : <Cible chiffrée mesurable d'ici fin trimestre>
• KR 1.2 : <…>
• KR 1.3 : <…>

Objectif 2 : <…>
• KR 2.1 : <…>
...
```

### Étape 5 — Aligne avec BP

Pour chaque OKR proposé, vérifie cohérence avec :
- Ramp BP (6→40→80→130 deals)
- Hypothèses 🔴 P1 BP Lab
- Phase 2 Compta (mission deadline 30 juin)
- Roadmap Organisation (missions 🔴 actuelles)

Si un OKR proposé contredit une hypothèse BP majeure → flag pour Comité Mentor.

### Étape 6 — Restitue le récap revue + propositions


## Anti-patterns

- ❌ **Ne fais pas 10 Objectifs par équipe** — 3-5 max, sinon dilution.
- ❌ **Ne valide pas un KR non-mesurable** — sans chiffre + date, c'est pas un KR.
- ❌ **Ne décide pas seul des OKR** — Paul + Julien tranchent, idéalement après input Philippe (BA).
- ❌ **Ne sous-évalue pas un échec** — un Objectif raté = enseignement, pas culpabilité.
- ❌ **Ne reconduis pas les OKR à l'identique** — si le contexte a changé, ajuste.
